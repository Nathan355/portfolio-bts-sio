C1 - Recensement et identification des ressources numériques

Guide de mise en œuvre : Gestion de parc avec GLPI
Objectif : Créer un inventaire automatisé pour ne plus rien perdre de son matériel.

1. Installation du "Cerveau" (Le Serveur)
Action : Préparer un serveur (Linux de préférence) avec une pile LAMP (Linux, Apache, MySQL, PHP).
Pourquoi : GLPI a besoin d'une base de données pour stocker les infos et d'un serveur web pour afficher l'interface.
Commande : sudo apt install apache2 mariadb-server php (Installe les composants nécessaires).
Téléchargez l'archive GLPI, décompressez-la dans /var/www/html et suivez l'assistant via votre navigateur.

2. Installation des "Espions" (Les Agents)
Action : Installer l'agent GLPI sur chaque machine à surveiller.
Procédure :
- Sur Windows : Lancer l'installateur .msi et renseigner l'URL du serveur (ex: http://votre-ip/glpi).
- Sur Linux : Utiliser la commande sudo apt install glpi-agent.
Résultat : La machine apparaît automatiquement dans l'onglet "Ordinateurs" de GLPI avec tous ses détails (RAM, CPU, Disque).

C7 - Collecte, suivi et orientation des demandes

Protocole : Transformer GLPI en centre de support (Ticketing)
Objectif : Centraliser les demandes pour qu'aucun collègue ne reste bloqué.

1. Structuration du terrain (Entités)
Action : Dans l'onglet "Administration > Entités", créer l'arborescence de l'entreprise (ex: RH, Comptabilité, Technique).
Pourquoi : Cela permet de trier les tickets automatiquement par département.

2. Cycle de vie d'un incident
Action : Définir le flux.
- Ouverture : L'utilisateur crée un ticket via son interface simplifiée.
- Traitement : Le technicien change le statut en "En cours" et ajoute un suivi.
- Clôture : Une fois résolu, passer en "Clos".
Exploitation : Utiliser le menu "Assistance > Statistiques" pour voir quels sont les problèmes récurrents et créer une mini-notice (FAQ) pour les résoudre plus vite la prochaine fois.

C11 - Référencement des services en ligne de l’organisation et mesure de leur visibilité

Guide : Publier un site et mesurer son succès
Objectif : Mettre un projet en ligne gratuitement et savoir qui le visite.

1. Mise en ligne (GitHub Pages)
Action : Créer un compte GitHub, créer un "Repository" et y glisser vos fichiers HTML/CSS.
Procédure : Aller dans "Settings > Pages" et choisir la branche "main". Votre site est en ligne en 30 secondes.
Pourquoi : C'est une solution professionnelle, gratuite et qui gère l'historique de vos modifications.

2. Pose du compteur (Google Analytics)
Action : Créer un flux de données sur Google Analytics.
Intégration : Copier le code (balise <script>) fourni par Google et le coller juste avant la balise </head> de votre fichier index.html.
Vérification : Ouvrez votre site, puis regardez l'onglet "Temps réel" sur Analytics. Si vous voyez "1 utilisateur", ça fonctionne !

C17 - Déploiement d’un service

Guide : La méthode Docker pour un déploiement infaillible
Objectif : Faire en sorte qu'une application marche partout de la même façon.

1. Création du conteneur
Action : Créer un fichier nommé Dockerfile.
Contenu type :

FROM nginx:latest
COPY ./mon-site /usr/share/nginx/html

Pourquoi : On dit à l'ordinateur : "Prends une base de serveur Web propre (Nginx) et mets mes fichiers dedans".

2. Lancement
Commande : docker build -t mon-app . (On fabrique l'image).
Commande : docker run -p 80:80 mon-app (On lance le service).
Avantage : Peu importe l'ordinateur de destination, si Docker est installé, l'application se lancera exactement de la même manière.
